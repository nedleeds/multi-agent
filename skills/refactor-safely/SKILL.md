사용자가 함수/클래스/모듈 이름 변경, 시그니처 수정, helper 추출, 파일 이동을 요청할 때 로드.

## 1. 영향 범위 먼저 파악 (코드 건드리기 전)
- 식별자를 `grep` — 언어에 맞춰 `type` 필터 사용 (`type='py'` 등).
- 문자열 참조도 `grep` — config, docs, tests, README 까지.
- callsite 수 집계. **20개 초과면 사용자에게 규모를 먼저 알리고 진행 여부 확인.**

## 2. 계약 이해
- 정의를 `read_file` 로 읽고 기록: 입력, 출력, 부수효과, 발생 가능한 예외.
- callsite 2-3개 읽어서 **어떻게 쓰이는지** 파악 (이름만 보지 말고 실사용 맥락).

## 3. 의존 순서대로 적용
- 정의 + 모든 callsite 를 한 번의 패스로 같이 수정.
- `edit_file` 은 **앞뒤 3줄 이상** 포함해 unique 하게 — 짧은 문자열 금지 (replace_all 오동작).
- 다수 파일 대량 rename: 배치 편집 + 각 단계 후 grep 으로 재확인.

## 4. 모든 callsite 갱신 검증 (CRITICAL)
- 단계 1의 grep 을 다시 실행.
- 옛 이름 히트 = 0 → 완료. 1개라도 남으면 계속 수정.
- 테스트 파일, 문자열 리터럴, 주석까지 확인.

## 5. 스모크 체크
- `bash`: 모듈 import, 기존 테스트 실행, 엔트리포인트 실행 중 하나.
- 실패하면 **패치에 패치를 쌓지 말고 되돌린 뒤 다시** 접근.

## 6. 보고
- Callsite 갱신: N 파일, M 줄.
- 검증: `<실행 명령>` 통과.
- 의도적으로 옛 이름 남겨둔 callsite (있으면 이유 명시).
